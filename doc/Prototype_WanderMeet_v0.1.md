# 旅聚 WanderMeet v0.1 低保真原型说明（小程序）

对应文档：

- `PRD_WanderMeet_v0.1_Beijing.md`
- `API_WanderMeet_v0.1.md`
- `DB_WanderMeet_v0.1.md`

视觉基线：

- 风格：年轻社交（偏克制）
- 主题：浅色
- 语言：全中文

---

## 1. 全局框架

## 1.1 底部 Tab（5 个）

- 首页
- 发现
- 发布
- 消息
- 我的

## 1.2 全局元素

- 顶部状态：定位城市（默认北京）+ 搜索/筛选入口
- 全局安全提示（弱提醒）：公开场所见面、勿私下转账、可一键举报
- 主 CTA 色：蓝色；危险操作：红色

## 1.3 页面跳转主链路

`首页活动流 -> 活动详情 -> 加入活动 -> 活动群聊 -> 到场反馈`

---

## 2. 页面一：首页活动流

## 2.1 目标

让用户 10 秒内找到“今天可参加”的活动。

## 2.2 结构（低保真）

```text
+--------------------------------------------------+
| [北京]                    [筛选] [搜索(可后置)]   |
|--------------------------------------------------|
| 快捷筛选: [今天] [明天] [附近3km] [咖啡] [徒步]   |
|--------------------------------------------------|
| 活动卡片 #1                                      |
| 标题: 周五晚国贸咖啡局                           |
| 时间: 今天 19:30 - 21:30                         |
| 地点: 国贸某咖啡馆 · 1.2km                        |
| 人数: 4/8  组织者: 已认证                        |
| [立即加入]                                       |
|--------------------------------------------------|
| 活动卡片 #2 ...                                  |
|--------------------------------------------------|
| 空状态: 暂无活动 -> [发布活动] [切换筛选]         |
+--------------------------------------------------+
| 首页 | 发现 | 发布 | 消息 | 我的                 |
+--------------------------------------------------+
```

## 2.3 关键组件

- 城市与筛选条：`dateRange`、`categoryId`、`maxDistanceKm`
- 活动卡片字段：标题、时间、地点、距离、已报名人数、组织者认证状态
- 空状态引导：鼓励发布活动，避免首页空窗

## 2.4 交互规则

- 点击卡片进入活动详情
- 点击立即加入：未登录先登录；登录后直接调用报名接口
- 报名成功后按钮状态变为“已加入”

---

## 3. 页面二：活动详情

## 3.1 目标

帮助用户快速判断是否参加，并明确活动规则与安全边界。

## 3.2 结构（低保真）

```text
+--------------------------------------------------+
| < 返回                                            |
|--------------------------------------------------|
| 标题: 周末奥森轻徒步                              |
| 标签: 徒步  免费  已认证组织者                     |
| 时间: 04/17 09:00 - 12:00                        |
| 地点: 奥林匹克森林公园南门                         |
| 地图缩略图                                        |
|--------------------------------------------------|
| 活动说明                                          |
| 休闲节奏，自备饮水...                             |
|--------------------------------------------------|
| 规则与安全                                        |
| ✓ 禁止骚扰  ✓ 禁止引流  ✓ 禁止不当内容            |
| 提示: 公共场所见面，勿私下转账                    |
|--------------------------------------------------|
| 成员预览: 组织者 + 3人                             |
|--------------------------------------------------|
| [举报]                          [立即加入 / 已加入] |
+--------------------------------------------------+
```

## 3.3 关键组件

- 活动核心信息区：标题、时间、地点、费用类型、人数进度
- 规则区：三条强规则可视化
- 成员预览：仅展示昵称与头像，保护隐私
- 举报入口：固定可见

## 3.4 交互规则

- 组织者本人展示“编辑活动/取消活动”
- 非成员点击“立即加入”后进入群聊
- 活动满员时按钮显示“人数已满”

---

## 4. 页面三：发布活动

## 4.1 目标

降低发起门槛，保证信息完整与风控前置。

## 4.2 结构（低保真）

```text
+--------------------------------------------------+
| 发布活动                                           |
|--------------------------------------------------|
| 标题*            [___________________________]    |
| 类目*            [咖啡 v]                         |
| 开始时间*        [2026-04-18 19:30 v]            |
| 结束时间         [2026-04-18 21:30 v]            |
| 地点名称*        [___________________________]    |
| 详细地址         [___________________________]    |
| 人数上限*        [ 8 v ]                          |
| 费用类型*        (免费)(AA)(固定金额)             |
| 活动说明*        [___________________________]    |
|                                                  |
| 规则勾选*                                         |
| [x] 禁止骚扰  [x] 禁止引流  [x] 禁止不当内容      |
|--------------------------------------------------|
| 未认证提示(条件显示): 发活动需完成认证 [去认证]    |
|--------------------------------------------------|
|                     [提交发布]                    |
+--------------------------------------------------+
```

## 4.3 关键组件

- 表单校验：必填项、时间合法性、人数上限范围
- 认证门槛：未认证不可提交
- 风控规则：必须勾选三项承诺

## 4.4 交互规则

- 提交后状态：`pending_review` 或 `published`
- 若待审核，给出“预计审核时间”提示
- 失败提示明确到字段级别

---

## 5. 页面四：活动群聊

## 5.1 目标

保障“成局”效率，降低沟通摩擦。

## 5.2 结构（低保真）

```text
+--------------------------------------------------+
| < 周末奥森轻徒步           [成员] [举报]           |
|--------------------------------------------------|
| 系统消息: 公开场所见面，注意安全                  |
|--------------------------------------------------|
| A: 大家好，明天南门见                            |
| B: 我会晚10分钟                                   |
| C: [图片]                                         |
|--------------------------------------------------|
| [输入消息...]                         [发送]      |
+--------------------------------------------------+
```

## 5.3 关键组件

- 聊天列表：文本/图片
- 顶部成员入口：查看活动成员
- 顶部举报：对用户/消息举报
- 系统安全消息：置顶提示

## 5.4 交互规则

- 仅活动成员可进入群聊
- 已拉黑用户消息对拉黑方不可见（服务端过滤）
- 违规消息支持长按举报

---

## 6. 页面五：我的（含认证状态）

## 6.1 目标

集中承载个人资料、认证、我的活动、风控记录。

## 6.2 结构（低保真）

```text
+--------------------------------------------------+
| 头像  旅人小王                                     |
| 标签: 数字游民 / 独行旅行                          |
|--------------------------------------------------|
| 认证状态: 已认证 / 审核中 / 未认证   [去认证]      |
|--------------------------------------------------|
| 我的活动                                          |
| [我发起的] [我参加的]                              |
|--------------------------------------------------|
| 安全中心                                          |
| [举报记录] [拉黑列表]                              |
|--------------------------------------------------|
| 设置                                              |
| [账号设置] [通知设置] [退出登录]                  |
+--------------------------------------------------+
```

## 6.3 关键组件

- 认证状态卡片：决定是否可发活动
- 我的活动二级页：按状态分组（进行中/已结束）
- 安全中心：增强信任感与透明度

---

## 7. 关键状态页（必须补齐）

- 登录前态：引导手机号登录
- 首页空状态：暂无活动，提供发布与筛选引导
- 报名成功态：跳转群聊
- 活动满员态：展示“人数已满”
- 网络失败态：重试按钮
- 账号限制态：展示限制原因和申诉入口（可后置为客服）

---

## 8. 文案基线（v0.1）

- 主按钮：`立即加入`、`发起活动`、`提交发布`
- 安全文案：`请优先选择公共场所见面`、`勿私下转账`、`发现违规可一键举报`
- 状态文案：
  - `审核中`
  - `已发布`
  - `人数已满`
  - `已取消`

---

## 9. 前端实现建议（便于快速落地）

- 先做低保真组件化：卡片、筛选条、状态标签、消息气泡、空状态
- 统一状态色：成功/警告/危险/中性
- 所有列表默认支持下拉刷新 + 触底加载
- 页面级埋点：
  - 首页曝光、卡片点击、报名点击、报名成功
  - 发布提交、审核结果
  - 举报提交
