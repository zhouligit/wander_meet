# WanderMeet 优化建议（2026-04-29）

基于当前已实现的小程序页面（首页/发现/发布/消息/我的）与后端接口现状，建议按「先体验与稳定，再增长」推进。

## P0（本周优先）

- 首页「今天/明天」数据准确性
  - 前端切换 tab 时明确传 `dateRange=today|tomorrow`
  - 后端 `GET /activities` 需要按北京时间做真实过滤，不只是排序
  - 目标：避免用户感知“筛选不准”

- 活动状态自动结束
  - 以 `endAt <= now` 作为已结束条件
  - 先在查询时动态判定，后续加定时任务回写 `ended`
  - 目标：避免历史活动混在进行中列表

- 消息已读一致性
  - 进入会话时立即调用 `PATCH /me/chats/:activityId/read`
  - 前端本地先更新未读数，后端成功后对齐
  - 目标：减少“看过仍红点”的投诉

- 发布页时间校验补强
  - 现有：结束时间晚于开始时间
  - 建议再加：开始时间不得早于当前时间（可容忍 5 分钟）
  - 目标：减少无效活动数据

## P1（2-3 周）

- 发现页接入附近能力
  - 使用 `GET /activities/nearby`
  - 增加半径筛选（3km/5km/10km）与距离展示
  - 目标：提升“就近参加”转化率

- 首页卡片信息密度优化
  - 增加“是否已报名”状态提示
  - 对齐后端报名状态字段
  - 目标：减少无效点击

- 消息列表排序统一
  - 按 `lastMessageAt desc`，无消息会话置底
  - 目标：符合 IM 使用习惯

- 我的页面统计口径统一
  - 参加数/发起数以服务端聚合为准
  - 目标：避免前后端显示不一致

## P2（增长与风控）

- 发布模板化
  - 咖啡/Citywalk/徒步预设默认时长与人数
  - 目标：提升发布转化

- 成行提醒
  - 达到阈值触发系统通知（复用通知接口）
  - 目标：提高到场率

- 举报/拉黑入口前置
  - 聊天页显式放置安全入口（复用 `/reports`、`/blocks`）
  - 目标：增强平台安全感

- 埋点漏斗建设
  - 曝光 -> 详情 -> 报名 -> 进群聊 -> 到场
  - 目标：为后续优化提供数据依据

## 页面最小改造建议

- 首页：优先修准「今天/明天」筛选逻辑
- 发现：接入 nearby 与距离展示
- 发布：补“开始时间不能早于现在”
- 消息：进会话即已读并即时更新红点
- 我的：统计统一以后端聚合结果展示
