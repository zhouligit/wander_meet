# 旅聚 WanderMeet PRD v0.1（北京首站版）

## 1. 产品目标

- 在北京验证：用户能在 24 小时内找到并参加线下活动。
- 首阶段不追求大而全，优先跑通成局率、安全、基础留存。

## 2. 产品定位与范围

### 2.1 产品定位

旅聚 WanderMeet 是面向独行旅行者、数字游民、独居青年的高意图线下活动平台，让用户在同城和旅途中快速找到靠谱的人一起参加线下活动。

### 2.2 核心承诺

今天就能找到人，一起做点事。

### 2.3 v0.1 功能范围（做）

- 注册登录、个人资料。
- 同城活动流（北京）。
- 创建/加入活动。
- 活动群聊。
- 举报/拉黑/封禁。
- 创建者认证（发活动需认证）。
- 基础消息通知。

### 2.4 v0.1 功能范围（不做）

- 复杂匹配算法。
- 1v1 私密匹配。
- 深度付费系统（仅预埋入口）。
- 多城市并发运营。

## 3. 目标用户与角色

### 3.1 目标用户

- 独行旅行者
- 数字游民
- 大量独居人群（同城高意图线下社交需求）

### 3.2 角色定义

- 参与者：浏览活动、加入活动、聊天、到场。
- 组织者：发起活动、管理报名、活动复盘。
- 运营管理员：审核活动、处理举报、封禁账号。

## 4. 核心用户流程

### 4.1 参与者流程

新用户注册 -> 完善资料 -> 浏览活动流 -> 加入活动 -> 进入活动群聊 -> 到场 -> 活动后反馈

### 4.2 组织者流程

认证通过 -> 发起活动 -> 审核通过 -> 活动曝光 -> 成团 -> 线下发生 -> 活动复盘

## 5. 信息架构（小程序主端）

- 首页：同城活动流（时间/距离/人数状态）
- 发现：活动分类（咖啡、citywalk、徒步、桌游、展览、夜跑）
- 发布：创建活动（时间、地点、人数、费用、规则）
- 消息：活动群聊、系统通知
- 我的：资料、认证、我的活动、举报记录、设置

## 6. 功能需求清单（v0.1）

### 6.1 账号与资料

- 手机号登录。
- 昵称、头像、个人标签（旅居/数字游民/周末出行等）。

### 6.2 活动流

- 卡片字段：标题、时间、地点、距离、已报名人数、组织者状态。
- 筛选：今天/明天、距离、类型。

### 6.3 创建活动

- 发活动前校验认证状态。
- 必填字段：标题、开始时间、地点、人数上限、活动说明。
- 规则勾选：禁止骚扰、禁止引流、禁止不当内容。

### 6.4 活动群聊

- 仅活动成员可见。
- 支持文本、图片。

### 6.5 风控安全

- 举报入口（活动页、聊天页、个人页）。
- 拉黑能力。
- 管理端封禁（手动）。

### 6.6 通知

- 报名成功、活动变更、举报处理、审核结果。

## 7. 风控策略（v0.1 必须落地）

### 7.1 认证策略

- 浏览与加入活动：手机号即可。
- 创建活动：必须完成认证（推荐实名 + 人脸，或实名 + 人工审核）。

### 7.2 违规处理分级

- 严重违规（骚扰、露骨内容、诈骗）：直接封禁。
- 一般违规：警告 -> 限制发言/发活动 -> 封禁。

### 7.3 安全提醒前置

在报名和活动详情页增加安全提示：公共场所见面、勿私下转账、可一键举报。

## 8. 冷启动运营方案（北京首站）

### 8.1 目标

前 4 周保证首页持续有“今天可参加”的活动供给。

### 8.2 供给三层模型

- S 层（官方种子）：每日 5-10 场模板活动。
- A 层（KOC 组织者）：招募 30-50 位，每人每周至少 2 场。
- B 层（商家共创）：10-20 家场地联合发活动（初期不收费）。

### 8.3 活动模板建议

- 下班咖啡局（2 小时）
- 周末 citywalk（3-4 小时）
- 夜跑社交局（1.5 小时）
- 桌游破冰局（2-3 小时）

## 9. 指标体系

### 9.1 北极星指标

24 小时成局率（活动发布后 24 小时内达最小成团人数并实际发生）。

### 9.2 一级指标

- 活动发布数/天
- 报名转化率（浏览 -> 加入）
- 到场率
- 举报率
- 7 日留存

### 9.3 预警指标

- 活动流空窗率（特定时段无可参加活动）
- 严重违规率
- 消息送达失败率

## 10. 技术实现建议（两人团队）

### 10.1 端策略

- 主端：微信小程序。
- App：后续补齐重度体验（推送、消息稳定性、长期留存）。

### 10.2 后端原则

- 统一 REST API。
- 核心模块：用户、活动、报名、聊天、举报、审核。
- 各端共用一套后端服务，先保证活动链路与消息稳定。

## 11. 6 周开发排期（建议）

- 第 1 周：原型确认、数据表设计、账号体系、基础框架
- 第 2 周：活动流 + 活动详情 + 报名
- 第 3 周：创建活动 + 认证校验 + 基础审核
- 第 4 周：活动群聊 + 通知 + 举报拉黑
- 第 5 周：简版管理后台 + 封禁流程 + 埋点看板
- 第 6 周：北京灰度上线 + 修复 + 运营联调

## 12. 待确认项（上线前必须拍板）

1. 认证方式：实名 + 人脸，或实名 + 人工审核
2. 首批活动类目保留数（建议 <= 4）
3. 活动默认人数上限（建议 6 或 8）
4. 是否允许纯线上活动（建议 v0.1 不允许）
5. 举报处理 SLA（建议 24 小时内）
6. 北京首批优先区域（朝阳/海淀/东城/西城排序）
